### 2026-04-03 22:35 Agent3 报告 #4：TTS 时间轴对齐方案

报告: `Report/report_4_timestamp_alignment.md`

**问题诊断**：
- TTS 与原音完全不对齐，根本原因是 `"。".join(translations)` 把所有翻译拼成一段连续音频
- TTS 从 0s 开始播放，但原音第一句可能在 24s 才开始（ASMR 有大量静默）
- `tts_delay_ms` 是全局偏移，只能临时对齐第一句，后续句子仍然漂移
- 数据流中时间戳断裂：ASR 返回 `{start, end, text}` → 翻译只保留纯文本 → TTS 无时间信息

**解决方案：逐句时间戳对齐**
- 核心方法 `build_aligned_tts()`: 逐句合成 TTS，按原音时间戳放置到时间轴上
- 新流程：ASR/VTT → `[{start, end, text}]` → 逐句翻译保留时间戳 → `[{start, end, text, translation}]` → 逐句 TTS → 对齐时间轴
- 输出 `tts_aligned.wav`（与原音等长），中文配音精确同步日文原音
- TTS 时长不匹配处理：短则静默等待，长则用 pytsmod 时间拉伸

**实施优先级**: P0 build_aligned_tts + Pipeline改造 > P1 Edge-TTS逐句并发 > P2 pytsmod拉伸 + 淡入淡出

### 2026-04-03 21:43 Agent3 报告 #3：VTT 智能流水线 + 多线程加速

报告: `Report/report_3_vtt_smart_pipeline.md`

**VTT 逻辑问题诊断：**

**多线程加速评估：**
- 单文件内：翻译/TTS 有数据依赖无法并行；模型预加载节省 3-5s
- 批量文件间：ThreadPoolExecutor 并行处理，推荐并行度 2
- GPU 管理：Semaphore 锁控制 GPU 并发访问，防 OOM
- 批量 8 文件：9.5min → 4.7min (2x 加速)
- 新建 `src/core/gpu_manager.py` GPU 资源管理器
