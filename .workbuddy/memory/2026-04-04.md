## Qwen3-TTS Bug 排查与修复

### 问题1: qwen_tts import 时进程静默退出
- **现象**: `import qwen_tts` 后整个 Python 进程以 exit code 0 退出，无任何错误信息
- **排查过程**: 逐层缩小范围 → `qwen_tts.core` → `tokenizer_25hz.modeling` → `vq.whisper_encoder` → 最终定位
- **根因**: 项目根目录下存在一个 `sox.py`（内容只有 `import sys; sys.exit(0)`），Python 模块查找机制会优先找到这个文件而非 site-packages 中的真正 sox 包
- **修复**: 删除 `d:\WorkSpace\AsmrHelper\sox.py`
- **排查方法**: monkey-patch sys.exit 捕获调用栈 + PowerShell stderr 显示 "SoX could not be found!"

### 问题2: Qwen3TTSModel API 已变更
- **现象**: `'Qwen3TTSModel' object has no attribute 'generate'`
- **根因**: qwen_tts 0.1.1 重构了 API
  - 旧: `model.generate(text, voice=..., speed=..., output_path=...)`
  - 新: `model.generate_custom_voice(text, speaker=..., language=..., instruct=...)` → `(List[np.ndarray], int)`
- **修复**: 更新 `src/core/tts/__init__.py` 中的 `_synthesize_custom_voice` 和 `_synthesize_from_cache`
- **注意**: 语言代码从 `"zh"` 改为 `"chinese"`，返回值改为 `(wavs, sr)` 需手动保存

### 核心发现
GUI 音色选择布局存在**架构级设计缺陷** 🔴

### 主要问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| Edge-TTS 音色选择器缺失 | 🔴 高 | 改造后只有 Qwen3-TTS 的 Tab，Edge-TTS 无音色选择 UI |
| 音色获取逻辑错误 | 🔴 高 | `_get_voice_info` 始终返回 Qwen3 格式音色，Edge-TTS 需要不同格式 |

### 修复建议
**方案 A** (推荐): 添加 Edge-TTS 音色选择器，使用 `QStackedWidget` 切换 Edge/Qwen3 音色 UI
**方案 B** (备选): 固定 Edge-TTS 使用默认音色，不在 GUI 中提供选择

### 优先级
1. 🔴 **P0**: 添加 Edge-TTS 音色选择器
