BUG-05：src/core/tts/__init__.py - synthesize_long_text 中的 asyncio.run 重复调用问题
文件: src/core/tts/__init__.py (第99行)

问题代码:

python
复制
for i, sent in enumerate(sentences):
    ...
    asyncio.run(self.synthesize_async(sent, str(temp_file)))
错误分析:
每个句子都调用一次 asyncio.run()，而 asyncio.run() 每次都会创建一个新的事件循环并关闭。在某些环境下（特别是已经存在运行中的事件循环，如 Jupyter 或部分 GUI 框架）会抛出 RuntimeError: This event loop is already running。
更好的方式是使用一个 asyncio.gather 统一处理或用同一个事件循环。

更正方案（简易）:

python
复制
async def _synthesize_all(self, sentences, temp_files):
    tasks = [self.synthesize_async(sent, str(tf)) for sent, tf in zip(sentences, temp_files)]
    await asyncio.gather(*tasks)

def synthesize_long_text(self, text: str, output_path: str) -> str:
    sentences = self._split_sentences(text)
    temp_files = [output_path.parent / f"temp_tts_{i}.mp3" for i in range(len(sentences))]
    asyncio.run(self._synthesize_all(sentences, temp_files))
    self._merge_audio(temp_files, output_path)
    for f in temp_files:
        f.unlink(missing_ok=True)
    return str(output_path)

BUG-08：src/mixer/__init__.py - mix_bilingual 方法的 ffmpeg filter 参数有误
文件: src/mixer/__init__.py (第165行)

问题代码:

python
复制

"[1:a][2:a][0:a]join=inputs=3:channel=5[out]",
错误分析:
join filter 的 channel 参数在 ffmpeg 中不是数字，应当使用通道布局字符串（如 channel_layout=5.1）。此处的 channel=5 语法不正确，会导致 ffmpeg 报错。此外，对于双语双轨场景通常只需要 stereo（2声道），不需要 5.1 声道布局。

更正方案（使用 amix 代替 join）:

复制

或直接使用 amerge:

python
复制
"-filter_complex",
"[0:a][1:a]amerge=inputs=2,pan=stereo|c0=c0|c1=c2[out]",
