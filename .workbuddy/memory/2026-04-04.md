## Report 7: VoiceTransl 翻译流程对比与迁移分析 (12:32)

**报告文件**: `Report/report_7_translation_migration.md`

### 核心发现

**VoiceTransl（GalTransl 引擎）vs AsmrHelper 翻译模块对比：**

| 维度 | VoiceTransl | AsmrHelper 现状 |
|------|-------------|-----------------|
| 请求粒度 | 10句/批，JSON 数组 | 1句/请求 |
| 重试机制 | 温度切换+断句重试，5次 | 无重试，直接保留原文 |
| 质量检测 | 残日/标点/换行/长度 | 无 |
| 翻译缓存 | 有（MD5 hash） | 无 |
| 字典系统 | 三层（pre/gpt/post） | 单层（system_prompt） |

**最大风险点**：翻译失败时 AsmrHelper 直接保留日文原文送入 TTS，会导致中文 TTS 引擎产出奇怪发音。

### 迁移优先级

- **Phase 1（1-2天）**：批量翻译(10句/批) + 重试机制 + 残日质量检测
- **Phase 2（3-5天）**：翻译缓存层 + 三层字典扩展
- **不迁移**：对话检测（ASMR 单人独白场景无意义）、Degeneration检测（云端 API 无此问题）

**API 费用**：迁移后 API 调用次数降低约 90%（100句 → 10次调用）。

## Report 14: 语音识别模块对比与迁移分析 (12:50)

**报告文件**: `Report/report_14_asr_migration.md`

### 核心结论

**VoiceTransl 的 ASR 模块对 AsmrHelper 没有迁移价值。**

原因：
1. VoiceTransl 面向正常音量视频/游戏配音，ASMR 需要轻声/气声特化适配
2. VoiceTransl 用外部进程调用 whisper.cpp，AsmrHelper 的 Python API 集成更先进
